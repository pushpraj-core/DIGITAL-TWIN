## Design Principles

1. **Modular Engines**: Each AI module is a standalone engine class that can be tested, replaced, or upgraded independently.

2. **Graceful Degradation**: ML models fall back to OpenCV/rules-based approaches when GPU or model weights are unavailable.

3. **Layered Architecture**: Presentation вҶ’ API вҶ’ Engine вҶ’ ML вҶ’ Data, with clear interfaces between each layer.

4. **Grid-Based Computation**: All spatial analysis operates on a unified terrain grid, enabling efficient numpy-based computations.

5. **Real-Time Recalculation**: Threat changes trigger cascading recalculations of risk heatmaps, routes, and visibility maps.

## Data Flow

1. **Upload** вҶ’ Image stored on disk, metadata in DB
2. **Analyze** вҶ’ CV pipeline generates terrain segments + grid
3. **Risk** вҶ’ Risk engine computes heatmap from terrain + threats
4. **Path** вҶ’ Movement planner builds graph from terrain + risk, runs pathfinding
5. **Observe** вҶ’ Observation engine computes visibility from elevation data
6. **Simulate** вҶ’ Replay engine generates timeline from route + risk
7. **Threat** вҶ’ Injection engine adds threats, triggers recalculation cascade
8. **What-If** вҶ’ Scenario engine modifies state, re-runs simulation pipeline
9. **AI** вҶ’ Assistant parses queries, calls appropriate engines, returns results
